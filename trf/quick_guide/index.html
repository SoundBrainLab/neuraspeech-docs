Preparing your data

This is a temporary quick guide for using the TRF module. More detailed documentation will be provided soon.

Step 1: Preparation

Once the environment is installed, make sure that you have the following:

  1. The stimuli. Have all the stimuli used in the EEG task in a folder. These should be individual audio sound tracks in .wav format.

  2. Preprocessed EEG files. Assume that you've completed all the preprocessing steps (e.g., filtering, ICA, resampling, etc.), epoched the data according to stimulus events (e.g., audio tracks), and saved as mne.Epochs objects in .fif format in the same folder. Each epoch should correspond to a stimulus sound track.

  3. A metadata file. A .csv file listing each EEG data file as well as its metadata. The most important things to have are a filename column listing the EEG filenames and a stim column listing the stimulus filenames (with file extension) for all epochs in the corresponding EEG file separated by semicolons (;). For example, a minimum metadta spreadsheet would look like:

    filename stim
    sub-01_task-alice.fif track1.wav;track2.wav;track3.wav;track4.wav;track5.wav

    In this example, the file sub-01_task-alice.fif has 5 epochs, and stim lists the corresponding stimulus filenames separated by ;. See misc/example_metadata.csv for an example.

Step 2: Get stimulus predictors

Load in the configuration file (default location: conf/conf.yaml) and genearate the gammatone envelope and onsets predictors from the stimuli:

import neuraspeech as ns
config = ns.trf.load_config()
ns.trf.make_gammatone_predictors(config)
This will create a folder called predictors and save the predictor files inside. Check the configuration file for details.

Step 3: Prepare EEG data

Load in the configuration file (default location: conf/conf.yaml) and prepare the EEG data:

import neuraspeech as ns
config = ns.trf.load_config()
ns.trf.prepare_eeg(config)
This will create a folder called eeg and save the prepared EEG files inside. Check the configuration file for details.

Step 4: Run the TRF model

The recommended way is to submit a job to a computing cluster and run the script with parallel processing enabled, espectially if you have many data files. Check the configuration files for details and see misc/example_job_script.sh for an example job script for Northwestern Quest.

Step 5: Analyze results

The results of each EEG data file will be as BoostingResult objects in .pickle formant under a folder named after the filename of the EEG data. You can load in the result files and extract prediction accuracy (correlation between true and predicted EEG), estimated TRFs, etc. See the documentation of the Eelbrain package.
